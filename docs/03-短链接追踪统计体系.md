# 03 短链接追踪统计体系

## 1. 统计目标
统计模块围绕三件事：

- 访问量：PV / UV / UIP。
- 画像维度：地区、操作系统、浏览器、设备、网络。
- 查询能力：单链接统计、分组统计、访问明细。

## 2. 采集链路（跳转 -> 消息 -> 落库）
统计并不是在跳转请求中同步落库，而是采用异步流水线：

1. 跳转成功后，组装 `ShortLinkStatsRecordDTO`。
2. `ShortLinkStatsSaveProducer.send(...)` 将记录写入 Redis Stream。
3. `ShortLinkStatsSaveConsumer` 消费消息，做幂等检查后执行实际落库。
4. 落库表包含：
  - `t_link_access_stats`（小时/星期维度）
  - `t_link_locale_stats`
  - `t_link_os_stats`
  - `t_link_browser_stats`
  - `t_link_device_stats`
  - `t_link_network_stats`
  - `t_link_access_logs`
  - `t_link_stats_today`
5. 同时回写 `t_link` 的总量统计字段（`total_pv/total_uv/total_uip`）。

## 3. 幂等与一致性
- 消费端引入消息幂等处理器，避免重复消费导致统计放大。
- 以短链为粒度加读锁，规避分组更新时的统计错位。
- 失败场景回滚幂等标识，交由重试机制补偿。

## 4. 查询链路
- 统计查询由 `ShortLinkStatsServiceImpl` 统一编排。
- 支持：
  - 单短链时间段统计
  - 分组时间段统计
  - 访问记录分页查询

管理端通过对应接口展示折线/分布图与明细列表。

## 6. 技术亮点
- 异步统计避免阻塞跳转主流程。
- 统计表设计按“总量 + 分维 + 明细”分层，兼顾效率与分析深度。
- 幂等 + 锁机制保证高并发下统计数据可信。
