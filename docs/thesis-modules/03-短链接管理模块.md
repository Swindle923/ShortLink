# 短链接管理模块（论文版）

## 1. 功能介绍
短链接管理模块覆盖短链全生命周期管理能力，包含分组管理、短链分页查询、更新、回收站恢复与下线治理等。该模块是运营侧的核心入口，决定了系统在“可维护、可治理、可审计”层面的工程质量。

管理模块的核心能力包括：
- 分组创建、重命名、排序、删除（软删除）
- 按分组查看短链列表与统计
- 编辑短链属性（有效期、描述等）
- 回收站恢复与彻底移除

## 2. 流程介绍
管理模块主流程可归纳为：
1. 用户登录后进入管理台，加载分组视图。
2. 前端以 `gid` 为主键组织查询和管理操作。
3. 管理操作通过 `admin -> aggregation -> project` 链路下发。
4. 后端在 DB 层执行归属校验（`username + gid + delFlag`）。
5. 管理结果回写前端列表与统计看板。

## 3. 具体原理
### 3.1 数据组织原理
- 分组采用 `t_group_unique` 与 `t_group` 双表设计：
- `t_group_unique` 负责 gid 全局唯一。
- `t_group` 负责业务字段（名称、排序、归属、删除标记）。
- 该拆分把“唯一性冲突处理”和“业务属性更新”解耦，提升维护性。

### 3.2 安全边界原理
- 更新、删除、排序统一使用“用户归属 + 状态”条件更新。
- 在 SQL 层约束操作范围，防止越权与脏写覆盖。
- 软删除策略保留历史状态，便于审计与恢复。

### 3.3 聚合视图原理
- 分组列表不是单表直出，而是“主数据 + 统计衍生数据”的组合视图。
- 管理台展示的短链数量由聚合服务统一计算，保证前端语义一致。

## 4. 设计思路
管理模块采用“以分组为业务坐标”的设计思想：
- 所有管理动作围绕 `gid` 展开，形成一致的数据治理边界。
- 通过软删除与回收站机制实现可逆变更，降低误操作风险。
- 通过聚合层屏蔽下游实现细节，保持接口稳定性。
- 通过条件更新将权限边界下沉到数据层，增强安全鲁棒性。

该模块的价值不仅在于“能改数据”，更在于让短链系统具备持续运营能力和工程治理能力。
