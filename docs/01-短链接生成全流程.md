# 01 短链接生成全流程

## 1. 功能目标
短链接生成模块要解决三个核心问题：

- 如何把长链接稳定映射成短码（`shortUri`）。
- 如何保证高并发下不重复、可重试、可追溯。
- 如何支持业务规则（分组、有效期、访问上限、描述）。

## 2. 端到端调用链路
从前端点击“创建”到数据库落盘，链路如下：

1. 前端创建弹窗提交请求（`/api/short-link/admin/v1/create`）。
2. `admin` 控制器接收并交给 `ShortLinkActualRemoteService`。
3. 单体模式下由 `aggregation` 的 `LocalShortLinkActualRemoteService` 直接编排 `project` 服务。
4. `project` 的 `ShortLinkServiceImpl.createShortLink()` 生成短码、校验规则、写入 `t_link` 与 `t_link_goto`。
5. 返回 `fullShortUrl + originUrl + gid` 给前端展示。

关键代码：

- `admin` 接口入口：`admin/.../controller/ShortLinkController.java`
- 单体本地“远程调用替代”：`aggregation/.../remote/LocalShortLinkActualRemoteService.java`
- 核心生成逻辑：`project/.../service/impl/ShortLinkServiceImpl.java`

## 3. 核心技术拆解
### 3.1 短码生成
- 通过生成策略得到 `shortUri`，再拼接域名得到 `fullShortUrl`。
- 同时写两张表：
  - `t_link`：业务主表（原链接、有效期、统计、配额等）。
  - `t_link_goto`：跳转映射表（短链到分组映射）。

### 3.2 防重复与并发安全
- 唯一索引兜底：数据库层确保重复短链不可写入。
- 布隆过滤器：用于降低缓存穿透与重复判断成本。
- 分布式锁版本（`create/by-lock`）：
  - 在高冲突场景下通过 Redisson 锁串行化关键段，减少重复失败。

### 3.3 业务规则注入
- 有效期模式：永久 / 指定日期。
- 分组归属：`gid` 决定管理范围和后续统计范围。
- 访问上限：`max_access_count` + `current_access_count`，用于后续跳转时配额消费。
- 描述字段：支持管理端检索和可视化。

### 3.4 失败处理
- `DuplicateKeyException` 转业务异常，保证用户可理解的失败反馈。
- 创建失败可重试，不会产生脏数据（依赖唯一约束和幂等逻辑）。

## 4. 数据落地结构
- 主表：`t_link`
  - `domain`
  - `short_uri`
  - `full_short_url`
  - `origin_url`
  - `valid_date_type`
  - `valid_date`
  - `max_access_count`
  - `current_access_count`
- 映射表：`t_link_goto`
  - `gid`
  - `full_short_url`

## 6. 本模块亮点
- 生成链路分层清晰（admin -> aggregation -> project）。
- 并发安全策略完整（唯一索引 + 锁 + 布隆过滤器）。
- 可运营字段齐全（分组、有效期、配额）并在管理端可回显。
