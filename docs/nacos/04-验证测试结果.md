# Nacos 集成验证测试结果

## 1. 验证环境
- 项目路径：`shortlink`
- 部署方式：Docker + 本地单体启动
- Nacos 镜像：`nacos/nacos-server:v2.2.3-slim`

## 2. 已完成验证

### 2.1 Nacos 容器部署验证
- 执行：`bash ./scripts/start-nacos.sh`
- 结果：成功启动，控制台地址可用（`http://127.0.0.1:8848/nacos`）

### 2.2 配置发布验证
- 执行：`bash ./scripts/nacos-publish-config.sh`
- 发布项：
  - `shortlink-common.yaml`
  - `short-link-aggregation.yaml`
- 结果：发布成功

### 2.3 配置读取验证
- 调用：
  - `GET /nacos/v1/cs/configs?dataId=shortlink-common.yaml&group=SHORTLINK_GROUP`
- 结果：成功读取，内容与发布一致。

### 2.4 动态白名单热更新验证（不重启）
- 用例：创建 `originUrl=https://example.com`
- 步骤与结果：
  1. `white-list.enable=false` 时创建成功（返回 `code=0`，生成短链如 `http://nurl.ink:8003/402l6L`）
  2. 动态改为 `white-list.enable=true` 后，同请求创建失败（返回 `A000001`，提示演示环境白名单限制）
  3. 再改回 `white-list.enable=false` 后，创建恢复成功（返回 `code=0`，生成短链如 `http://nurl.ink:8003/1m0d9r`）
- 结论：Nacos 配置变更已被应用运行时感知并生效。

### 2.5 动态风控阈值热更新验证（不重启）
- 用例：`/api/short-link/admin/v1/user/has-username?username=admin`
- 步骤与结果：
  1. 动态设置 `flow-limit.max-access-count=1`、`time-window=5`
  2. 连续 3 次请求结果：
     - 第 1 次：成功（`code=0`）
     - 第 2、3 次：被限流（`A000300`，系统繁忙）
  3. 动态恢复 `max-access-count=20`、`time-window=1`
- 结论：策略规则阈值实现动态下发和实时生效。
