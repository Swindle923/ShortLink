# 02 短链接跳转全流程

## 1. 功能目标
跳转模块是系统实时流量入口，要求：

- 快：优先缓存命中，降低数据库压力。
- 准：有效期、启用状态、配额都要正确判断。
- 稳：高并发下统计与跳转要解耦，不拖慢用户重定向。

## 2. 跳转主链路
入口在 `project` 的重定向接口，核心由 `ShortLinkServiceImpl.restoreUrl(...)` 承担：

1. 根据 `domain + shortUri` 得到 `fullShortUrl`。
2. 查询 Redis 缓存：
  - 命中目标长链接：直接 302。
  - 命中“空值标记”：快速返回 404，防穿透。
3. 缓存未命中时查库（`t_link_goto` -> `t_link`）。
4. 校验状态：
  - 是否启用
  - 是否过期
  - 是否超配额（`max_access_count/current_access_count`）
5. 校验通过后：
  - 回填缓存
  - 执行重定向
  - 异步上报统计消息（PV/UV/UIP/设备/网络等）

## 3. 配额控制细节（本次核心增强）
- 每次跳转尝试会执行原子消费：
  - `current_access_count = current_access_count + 1`
  - 条件：`current_access_count < max_access_count`
- 若消费失败（配额耗尽）：
  - 自动将短链置为不可用
  - 清理可跳转缓存
  - 写入空值缓存，避免持续打库
- `max_access_count` 为空表示“不限”，直接放行。

## 4. 缓存与防穿透设计
- 正向缓存键：短链 -> 原始 URL
- 空值缓存键：短链不存在/不可用的标记
- 布隆过滤器：减少不存在短链的数据库探测

## 5. 可靠性与性能
- 跳转路径“读多写少”，尽量走 Redis，降低 DB 热点。
- 统计写入采用消息流异步化，避免把用户重定向阻塞在统计逻辑上。
- 对分组变更等会用读写锁与缓存失效策略保持一致性。

## 7. 本模块亮点
- 缓存优先 + 防穿透 + 配额原子扣减。
- 跳转与统计解耦，用户体验和统计完整性兼顾。
- 过期、禁用、超限三类状态统一治理，行为可预期。
