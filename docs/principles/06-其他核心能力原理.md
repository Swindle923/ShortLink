# 其他核心能力原理（论文叙事版）

如果把短链接系统只理解成“创建与跳转”两件事，最终很容易得到一个功能上可用、工程上脆弱的系统。真实业务里，长期稳定运行更依赖那些看似次要、实际决定系统边界的治理能力。当前项目把这些能力集中在访问配额、回收站语义和权限控制三个方面，它们共同构成了系统的“约束层”，让短链接平台不至于在规模增长后失控。

访问配额是最直接的治理机制。在数据模型中，`t_link` 同时保存 `max_access_count` 与 `current_access_count`，前者定义理论上限，后者记录已消耗额度。关键不在字段存在，而在扣减动作的并发语义。`ShortLinkServiceImpl#tryConsumeAccessQuota` 并没有采用先查后改，而是调用条件更新 SQL：仅在当前计数小于上限时才允许自增。这个实现保证了配额消费的原子性，避免了并发访问下“多个线程同时通过校验”导致的超卖。配额耗尽后，系统还会执行状态收敛动作，把短链禁用并同步清理缓存，这样后续请求会快速失败，不会继续反复冲击数据库。

回收站机制则负责“可恢复删除”与“安全下线”的平衡。`RecycleBinServiceImpl` 中的三个动作分别对应三种状态迁移：保存到回收站会把 `enableStatus` 置为不可用，恢复会重新置为可用，移除则通过 `delFlag + delTime` 进入逻辑删除状态。它的设计重点在于状态与缓存联动，不同迁移动作都会对 `goto` 或 `is-null` 缓存进行清理或重置，避免出现管理台显示已恢复但用户仍访问失败，或者管理台已下线但缓存仍可跳转的语义错位。换句话说，回收站不是“多一张页面”，而是完整生命周期管理的一部分。

权限体系看起来更简单，实则直接影响管理面的安全边界。系统在 `t_user` 上引入 `role` 字段，并在 `UserAuthenticationFilter` 中把鉴权与鉴权后的授权分开处理。普通接口先做 `Username + Token` 会话校验，管理员接口再做角色判断；如果历史账号 `admin` 没有显式角色字段，代码还提供了兼容分支，按管理员处理。这个实现虽然轻量，但核心思想是正确的：认证解决“你是谁”，授权解决“你能做什么”，二者必须解耦。对于后台系统来说，这种最小权限模型可以有效减少误操作和越权风险。

此外，系统还在用户行为入口增加了流量风控过滤器，通过 Lua 脚本在 Redis 侧执行时间窗计数。这意味着风控决策不依赖单机内存，不会因为多实例部署而失真。它与鉴权过滤器叠加后，形成了“先识别身份、再限制频率、最后执行业务”的防护顺序。这样的顺序在工程上非常重要，因为它把最便宜、最通用的防护前置，降低了后端业务方法暴露在异常流量下的概率。

从系统工程视角看，这几类能力有一个共同特征：它们并不直接提升“功能炫酷程度”，却显著提升“系统可治理性”。配额让资源消耗可控，回收站让状态演进可逆，RBAC 与风控让管理入口可防护。正是这些约束层能力，使短链接系统能够从“课程作业级功能实现”走向“可上线运行的工程系统”。在毕业设计语境下，这部分内容是非常加分的，因为它体现了你对软件系统生命周期和运维现实的真实理解，而不仅仅是 API 层面的功能堆叠。

如果把这些治理能力进一步落到代码级实现，访问配额最核心的判断其实不在 Java 里，而在 Mapper 的条件更新语句中。只有当短链状态可用且 `current_access_count < max_access_count` 时，数据库才允许本次访问计数加一，这保证了并发下不会超卖额度。

```sql
<update id="incrementCurrentAccessCount">
    UPDATE t_link
    SET current_access_count = current_access_count + 1
    WHERE gid = #{gid}
      AND full_short_url = #{fullShortUrl}
      AND del_flag = 0
      AND enable_status = 0
      AND (max_access_count IS NULL OR current_access_count < max_access_count)
</update>
```

服务层收到影响行数后，会立刻做状态收敛。成功则继续放行，失败则禁用短链并清缓存，避免后续请求继续打到昂贵路径。

```java
private boolean tryConsumeAccessQuota(String gid, String fullShortUrl) {
    int updated = baseMapper.incrementCurrentAccessCount(gid, fullShortUrl);
    if (updated > 0) {
        return true;
    }
    baseMapper.disableByAccessLimit(gid, fullShortUrl);
    stringRedisTemplate.delete(String.format(GOTO_SHORT_LINK_KEY, fullShortUrl));
    stringRedisTemplate.opsForValue().set(String.format(GOTO_IS_NULL_SHORT_LINK_KEY, fullShortUrl), "-", 30, TimeUnit.MINUTES);
    return false;
}
```

回收站的状态迁移同样是硬编码在服务方法里的，而不是抽象概念。下面三个方法分别对应“下线”“恢复”“逻辑删除”，并且每一步都和缓存处理绑定，确保控制台状态与跳转状态一致。

```java
public void saveRecycleBin(RecycleBinSaveReqDTO requestParam) {
    baseMapper.update(ShortLinkDO.builder().enableStatus(1).build(), updateWrapper(...enableStatus=0...));
    stringRedisTemplate.delete(String.format(GOTO_SHORT_LINK_KEY, requestParam.getFullShortUrl()));
}

public void recoverRecycleBin(RecycleBinRecoverReqDTO requestParam) {
    baseMapper.update(ShortLinkDO.builder().enableStatus(0).build(), updateWrapper(...enableStatus=1...));
    stringRedisTemplate.delete(String.format(GOTO_IS_NULL_SHORT_LINK_KEY, requestParam.getFullShortUrl()));
}

public void removeRecycleBin(RecycleBinRemoveReqDTO requestParam) {
    ShortLinkDO del = ShortLinkDO.builder().delTime(System.currentTimeMillis()).build();
    del.setDelFlag(1);
    baseMapper.update(del, updateWrapper(...enableStatus=1...delTime=0...));
}
```

权限控制部分同样有清晰代码边界。`UserAuthenticationFilter` 先筛选管理台路径，再做公开接口豁免，然后校验 `Username + Token`，最后在管理员接口上做角色判定。这个顺序保证了高频请求先走便宜判断，昂贵验证后置。

```java
if (!requestUri.startsWith("/api/short-link/admin/v1/")) {
    filterChain.doFilter(request, response);
    return;
}
if (isPublicApi(requestUri, httpServletRequest.getMethod())) {
    filterChain.doFilter(request, response);
    return;
}
String username = decodeHeader(httpServletRequest.getHeader("Username"));
String token = httpServletRequest.getHeader("Token");
Object loginUserJson = stringRedisTemplate.opsForHash().get(USER_LOGIN_KEY + username, token);
if (loginUserJson == null) {
    returnJson(httpServletResponse, HttpServletResponse.SC_UNAUTHORIZED, "登录已失效，请重新登录");
    return;
}
if (isAdminApi(requestUri) && !Objects.equals(role, UserRoleEnum.ADMIN.name())) {
    returnJson(httpServletResponse, HttpServletResponse.SC_FORBIDDEN, "无权限执行该操作");
    return;
}
```

因此这几项“其他能力”并不是外围描述，而是都有可定位、可验证、可回归测试的实现点。你在论文里只要把这些代码语义讲透，评审就能看到这不是概念拼贴，而是完整工程落地。
