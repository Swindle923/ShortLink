# 04 短链接分组管理

## 1. 为什么要做分组
短链接系统本质是管理系统。分组是“多业务场景可管理”的基础能力，用于：

- 按项目/活动/渠道隔离短链。
- 限制用户可操作范围（列表、统计、回收站）。
- 支撑后续报表按分组聚合。

## 2. 分组能力清单
当前分组能力已经覆盖完整生命周期：

- 新增分组
- 查询分组列表
- 修改分组名称
- 删除分组
- 分组排序
- 默认分组自动创建（用户注册后）

接口入口在 `admin` 的 `GroupController`，业务实现位于 `GroupServiceImpl`。

## 3. 分组与短链关系
- 创建短链时必须指定 `gid`。
- 分页查询短链按 `gid` 过滤。
- 回收站分页按 `gidList` 过滤。
- 统计查询支持按单个 `gid` 聚合。

这意味着分组不仅是“标签”，而是查询与权限边界。

## 4. 分组管理的稳定性策略
- 通过用户上下文拿当前账号，避免跨用户操作分组。
- 分组变更后联动缓存与读写锁策略，保证跳转与统计一致。
- 分组排序支持批量更新，满足运营台快速调整展示顺序。

## 6. 管理系统角度的价值
- 让“短链”从单点工具变成可运营资产。
- 让统计/回收站/权限都以分组为中心组织，后续扩展成本低。
- 为后续多租户或组织层级打下结构基础。
